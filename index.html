专门为 Elasticsearch 设计的工具

  • ES Pref & Elasticsearch-stress-test

¶官方开源压测工具ES Rally

  • 基于python3开发
  • 性能测试结果比较： https://elasticsearch-benchmarks.elastic.co

¶安装

如果git版本低于1.9，需要升级git版本

yum install http://opensource.wandisco.com/centos/7/git/x86_64/wandisco-git-release-7-2.noarch.rpm -y
yum update git -y

优化.md
2020年05月14日 104 字 大概 1 分钟

1、每个es节点RAM不超过32G；

2、每个分片数据量30GB左右，不要超过50GB

每个分片(shard)本质是一个lucene 索引，Lucene硬性限制每个index文档最大数量为2的31次方，即20亿。随着 shard 体积的增大，其查询效率会下降，而且数据迁移和恢复的成本也会增高。官方建议单个 shard 大小不要超过 50GB
